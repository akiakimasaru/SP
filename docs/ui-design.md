# 営業支援AIエージェントプラットフォーム - UI設計

## 1. UI設計原則

- **シンプルで直感的**: 営業担当が迷わず操作できる
- **情報の可視化**: エージェントの状態・実行結果を視覚的に表示
- **レスポンシブ**: デスクトップ・タブレット・モバイル対応
- **アクセシビリティ**: WCAG 2.1 AA準拠

## 5. 共通コンポーネント

### 5.1 エージェントカード
- エージェント名
- ステータス表示（実行中/待機中/エラー）
- 最終実行日時
- 実行回数
- クイックアクションボタン

### 5.2 ステータスバッジ
- 成功: 緑色
- 警告: オレンジ色
- エラー: 赤色
- 待機中: グレー

### 5.3 ローディングインジケーター
- スピナー
- プログレスバー
- スケルトンローディング

### 5.4 通知バナー
- 成功通知: 緑色
- 警告通知: オレンジ色
- エラー通知: 赤色
- 情報通知: 青色

### 5.5 モーダルダイアログ
- 確認ダイアログ
- フォームダイアログ
- 詳細表示ダイアログ

## 6. レスポンシブデザイン

### 6.1 ブレークポイント
- Mobile: < 768px
- Tablet: 768px - 1024px
- Desktop: > 1024px

### 6.2 モバイル対応
- サイドバーをドロワーに変更
- カードレイアウトを縦並びに
- タブをアコーディオンに変更

## 7. アニメーション

- ページ遷移: フェードイン/アウト
- ローディング: スピナー/プログレスバー
- 通知: スライドイン/アウト
- ホバーエフェクト: シャドウ・色変更

## 8. アクセシビリティ

- キーボードナビゲーション対応
- ARIAラベル設定
- コントラスト比確保
- フォーカスインジケーター
- スクリーンリーダー対応
